    Native UI programming on Windows using C++: Part 1

    Setting up project to explore win32 GUI APIs

    Ajay 2023-05-05

    In next few posts we will explore fundamental APIs provided by Microsoft Windows to create user interfaces. First of all we will setup our environment and project structure. We will use C++ as programming language and Visual Studio 2022 as IDE. I have Microsoft Visual Studio Community 2022 (64-bit) Version 17.2.6 installed on my machine. This is a freely available edition of Visual Studio which can be downloaded from this link. When you run the installer, you should select Desktop development with C++ workload for installation.

    Once installation is done, launch visual studio and create a new project. There are different types of projects for different requirements. We will start with an empty project. Empty project does not provide any starter code. It just contains file related to project and solution (A solution is nothing but a collection of projects. A project contains code and resources and generates an artifact like a lib/dll/exe when compiled). I have created a project called EmptyWindow and I have named my solution NativeUI. You can see the state of solution at this point here

    Once your solution is in same state as shown in above github link, you can quickly validate if everything is working fine by writing a 'hello world' program. To do this, right click on your project under solution explorer window in visual studio. Select Add > New Item from right click menu and add a new C++ file named main.cpp.

    Add following code in main.cpp:

    Try to build and run the project. It should launch a console window and print the string "hello world!". Now we have a working IDE and a properly setup solution. Current state of the solution is here

    But we do not want to create a console app (i.e. a command line app), we want to create a Graphical User Interface based app and we need to tell this to Visual Studio. We do this by making following changes in the project properties. Project Properties dialog can be accessed by right clicking on project under solution explorer and selecting Properties from the menu. Once Project Properties dialog is open, make following changes (make sure you make these changes for all relevant configurations and platforms which are controlled by two dropdowns on top of the window):

    • Under Configuration Properties > General > C++ Language Standard select ISO C++ 20 Standard. This is not really required but it is good to use the latest and greatest version of the language.
    • Under C/C++ > PreProcessor > PreProcessor Definitions remove _CONSOLE if present and add _WINDOWS.
    • Under Linker > System > SubSystem select Windows

    Once these changes are done, our previous "hello world!" code will not work.So we can remove that and now we are ready to start adding code for creating an empty window. But we will do that in next post. At this point, your solution should look like this
